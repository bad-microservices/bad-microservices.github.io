    API Reference¶

    A Helper Library to configure AMQP Connections.

    This Module contains the AMQPConfig Class which can be helpfull to connect to AMQP Brokers.

    class amqp_helper.AMQPConfig(host='localhost', port=5672, username='guest', password='guest', vhost='/', connection_name=None, message_lifetime=60, ca_file=None)[source]¶

    Config class for AMQP Connections

    Variables:¶
    host : str, optional¶

    Host or IP of the AMQP Server. Defaults to "localhost"

    port : int, optional¶

    Port to use. Defaults to 5672.

    username : str, optional¶

    Username used for connecting to the AMQP Server. Defaults to "guest".

    password : str, optional¶

    Password used for connecting to the AMQP Server. Defaults to "guest".

    vhost : str, optional¶

    Vhost which we want to connect to. Defaults to "/".

    connection_name : str, optional¶

    Name to register the connection with. Defaults to None.

    message_lifetime : int, optional¶

    Expiry Time of messages in seconds. Defaults to 60.

    ca_file : str, optional¶

    If you want to use an SSLContext for connecting specify the path to the ca File here. Defaults to None.

    property ssl¶

    True if you specified an ca_file.

    Type:¶

    bool

    property ssl_context¶

    if you specified an ca_file will return an SSLContext. If not it will return None.

    Type:¶

    ssl.SSLContext | None

    class amqp_helper.AMQPLogHandler(amqp_config)[source]¶
    emit(record)[source]¶

    Emit a record.

    If a formatter is specified, it is used to format the record. The record is then written to the stream with a trailing newline. If exception information is present, it is formatted using traceback.print_exception and appended to the stream. If the stream has an ‘encoding’ attribute, it is used to determine how to do the output to the stream.
