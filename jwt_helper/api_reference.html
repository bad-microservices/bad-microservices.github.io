    class jwt_helper.SignMethod(value)[source]¶

    Class which represents different Signing Methods for JWT decoding

    This class contains several Values each representing an algorithm used for encoding and decoding JWTs.

    Supported Methods:
    • Symetric

      • HS256

      • HS384

      • HS512

    • Asymetric

      • RS256

      • RS384

      • RS512

    class jwt_helper.Issuer(name, secret, methods)[source]¶

    This class represents an Issuer

    An Issuer is an auth-server which creates JWT Tokens. We list supported Methods for validating there cookies and the Secret belonging to it here. We have a list of supported methods here so Attackers cant swap the signing method around (for example switching from HS to RS Signature and using the Public Key as HMAC secret)

    name¶

    The name of the issuer which is also included in the JWT Header

    Type:¶

    str

    secret¶

    either the symetric HMAC secret or the Public key used to verify the Signature

    Type:¶

    bytes | string

    methods¶

    The list of supported Signin Methods for the specified Issuer

    Type:¶

    list[SignMethod]

    verify_jwt(token)[source]¶

    Function to check if a token is valid

    This Function checks a JWT Token for it’s validity. The Tokens ‘iss’ Header entry gets used to determine the Issuer from our Issuers list.

    Parameters:¶
